\subq{c} What non-matmul kernels account for non-trivial CUDA runtime in the forward pass?

Besides matrix multiplications, non-trivial forward-pass CUDA time came from elementwise kernels such as division, \texttt{where}, add, multiply, exponentiation, sigmoid, copy/concat kernels, and reduction kernels used for max, sum, and normalization.
For the XL model at context length $1024$, matrix multiplications accounted for $86.8\%$ of GPU kernel time, leaving about $13.2\%$ for these non-matmul kernels.
\end{answer}

For the small model at context length $1024$, the matmul fraction dropped from $69.6\%$ in forward-only to $64.7\%$ in the full forward+backward+optimizer step.
For the XL model, it dropped from $86.8\%$ to $82.3\%$. This happens because backward and AdamW introduce many additional pointwise/vectorized kernels, so non-matmul work becomes a larger fraction even though GEMMs still dominate.
\end{answer}

\subq{e} Compare softmax runtime to attention matmul runtime in the forward pass. How does the runtime ratio compare to the FLOP ratio?
\deliverable{A 1--2 sentence response.}
\begin{answer}
The softmax operation did not appear as one fused softmax kernel; it was decomposed into max reductions, exponentials, sum reductions, and division.
For the XL model at context length $1024$, these softmax-like kernels were roughly $4$--$5\%$ of forward GPU kernel time, while matmuls were $86.8\%$. Thus matmul runtime was about an order of magnitude larger, much less extreme than the FLOP gap, because softmax is memory-, reduction-, and special-function-heavy and runs less efficiently than Tensor Core GEMMs.
\end{answer}

  \includegraphics[width=\linewidth]{nsys.png}

For the xl model, I measured one TransformerBlock using Nsight Systems with CUDA memory
profiling enabled and PyTorch NVTX labels. Nsight reported total CUDA memory around 19 GiB,
while \texttt{torch.cuda.memory\_allocated()} reported a little over 13 GiB. I used the
PyTorch allocated-memory deltas for saved residual tensors because \texttt{memory\_allocated()}
tracks live PyTorch tensor memory, whereas Nsight also includes allocator caching,
reserved blocks, CUDA runtime memory, and temporary library workspaces. The total saved
forward memory for one TransformerBlock was about 180 MiB. The five largest contributors were
\texttt{ffn.w3 aten::linear} at 20.00 MiB (11.09\%),
\texttt{ffn.w1 aten::linear} at 20.00 MiB (11.09\%),
\texttt{ffn.silu aten::silu} at 20.00 MiB (11.09\%),
\texttt{ffn.gate aten::mul} at 20.00 MiB (11.09\%), and
\texttt{attn.softmax aten::softmax} at 16.19 MiB (8.97\%).

During the backward pass, the saved forward tensors are freed while gradient tensors are
allocated at the same time. In the Nsight trace, the memory usage decreases by about
180 MiB across a single \texttt{TransformerBlock} backward range, which approximately
matches the amount of memory saved during the forward pass. This indicates that the memory
freed from activation tensors largely cancels out the memory allocated for gradients, due
to allocator reuse and overlapping lifetimes. As a result, the net change in memory usage
does not directly reflect the gradient tensor size.

However, we still expect the gradient memory to be on the order of the parameter size of
the block, since each parameter produces one gradient tensor of the same shape. This is
much smaller than the activation memory saved during the forward pass, which scales with
batch size and sequence length. The observed behavior therefore matches expectations:
activation memory dominates in the forward pass, while gradient memory is comparatively
smaller and masked by memory reuse during backward.
\end{answer}

\problem{gradient\_checkpointing}{4}

\subq{a} For a Transformer of $N$ identical blocks, what checkpointing strategy minimizes peak activation memory (ignoring compute)? Give asymptotic peak memory and compute as a function of $N$, plus a short code sketch.
\deliverable{A 3--5 sentence description plus a short code sketch.}
\begin{answer}
  If we only care about memory and can sacrifice compute as much as we want, we can use a fully nested (recursive) checkpointing strategy: for block $i$, checkpoint the entire prefix $0,\dots,i-1$, then execute block $i$ normally. During backward, each checkpoint recomputes its prefix just-in-time, so at any moment only $O(1)$ block residuals are alive.
  Thus, the peak activation memory is
  \[
  M(N) = \Theta(1),
  \]
  which is optimal up to constants. However, the compute cost is
  \[
  C(N) = \Theta(N^2),
  \]
  since prefixes are recomputed repeatedly:
  \[
  N + (N-1) + \cdots + 1 = \Theta(N^2).
  \]

  A sample code looks like
\begin{algorithm}[H]
\caption{Recursive checkpointed prefix evaluation}
\begin{algorithmic}[1]
\Procedure{RunPrefix}{$\textit{blocks}, i, x$}
  \If{$i = 0$}
    \State \Return $\textit{blocks}[0](x)$
  \EndIf
  \State $h \gets \textsc{Checkpoint}(\lambda y.\ \textsc{RunPrefix}(\textit{blocks}, i-1, y),\ x)$
  \State \Return $\textit{blocks}[i](h)$
\EndProcedure
\end{algorithmic}
\end{algorithm}
\end{answer}

\subq{b} For the xl model (batch size 4, seq len 2048) with a budget of \emph{one} step of recomputation (no nesting), what is the best checkpointing block size? Validate with peak-memory profiles, comparing the next-smaller and next-larger block sizes.
\deliverable{A 3--5 sentence description plus measured peak memory.}
\begin{answer}
  For the xl config with $N=32$, the best one-step strategy is to wrap the
entire stack of 32 Transformer blocks in a single checkpoint call. In the
ideal activation-only model, checkpointing 32 blocks instead of 31 blocks saves
only one extra residual stream tensor:
\[
4 \cdot 2048 \cdot 1600 \cdot 2 \text{ bytes}
\approx 25 \text{ MiB}
\approx 0.024 \text{ GiB}.
\]
This is tiny relative to the full training-step peak, which is dominated by
persistent model parameters, gradients, optimizer state, CUDA allocator
reserved memory, and temporary library workspaces. Consistent with this, my
profiles showed essentially no measurable peak-memory change between the
largest checkpoint size and the next-smaller checkpoint size. Thus I used one
checkpoint around all 32 blocks, but the measured peak memory was effectively
unchanged because the saved activation difference was too small compared with
the optimizer-dominated baseline.
\end{answer}

\problem{data\_parallel\_calcs}{3}
(FP16 weights/activations, ignore non-matmul FLOPs.)

\subq{a} Backward-pass FLOPs with $N_{\mathrm{DP}}$ data parallelism.
\deliverable{Answer in terms of $B, D, D_{\mathrm{FF}}, N_{\mathrm{DP}}$ with one-sentence justification.}
\begin{answer}
  The backward pass costs
  \[
    \frac{12 B D D_{\mathrm{FF}}}{N_{\mathrm{DP}}}
  \]
  FLOPs per device. There are six matmuls in the FFN backward pass, each using only the local batch shard of size \(B/N_{\mathrm{DP}}\).
\end{answer}

\subq{b} Backward-pass communication time with $N_{\mathrm{DP}}$ data parallelism.
\deliverable{Answer in terms of subset of $B, D, D_{\mathrm{FF}}, N_{\mathrm{DP}}, W$ with one-sentence justification.}
\begin{answer}
  The communication time is
  \[
    2\frac{N_{\mathrm{DP}}-1}{N_{\mathrm{DP}}}\frac{6 D D_{\mathrm{FF}}}{W}
    =
    \frac{12 (N_{\mathrm{DP}}-1) D D_{\mathrm{FF}}}{N_{\mathrm{DP}} W}.
  \]
  We all-reduce the three FP16 weight gradients, whose total size is \(3 D D_{\mathrm{FF}}\) elements \(=6D D_{\mathrm{FF}}\) bytes.
\end{answer}

\subq{c} How large can $N_{\mathrm{DP}}$ grow before communication bottlenecks?
\deliverable{Inequality with $N_{\mathrm{DP}}$ on one side; right-hand side in terms of subset of $B, D, D_{\mathrm{FF}}, C, W$.}
\begin{answer}
  We need communication time to stay below compute time:
  \[
    \frac{12 (N_{\mathrm{DP}}-1) D D_{\mathrm{FF}}}{N_{\mathrm{DP}} W}
    \le
    \frac{12 B D D_{\mathrm{FF}}}{N_{\mathrm{DP}} C}.
  \]
  After canceling common factors, this gives
  \[
    N_{\mathrm{DP}} \le 1 + \frac{B W}{C}.
  \]
\end{answer}


\problem{fsdp\_calcs}{3}

\subq{a} FLOPs for the forward and backward pass with $N_{\mathrm{FSDP}}$.
\deliverable{Two answers in terms of $B, D, D_{\mathrm{FF}}, N_{\mathrm{FSDP}}$ with two one-sentence justifications.}
\begin{answer}
  The forward pass costs
  \[
    \frac{6 B D D_{\mathrm{FF}}}{N_{\mathrm{FSDP}}},
  \]
  since each device runs the usual three FFN matmuls on a \(B/N_{\mathrm{FSDP}}\) batch shard. The backward pass costs
  \[
    \frac{12 B D D_{\mathrm{FF}}}{N_{\mathrm{FSDP}}},
  \]
  for the same reason as DP: the local backward matmuls only see the local batch shard.
\end{answer}

\subq{b} Communication time for the forward and backward pass with $N_{\mathrm{FSDP}}$.
\deliverable{Two answers in terms of a subset of $B, D, D_{\mathrm{FF}}, N_{\mathrm{FSDP}}, W$.}
\begin{answer}
  The forward pass all-gathers the three FP16 weights, so it costs
  \[
    \frac{N_{\mathrm{FSDP}}-1}{N_{\mathrm{FSDP}}}\frac{6 D D_{\mathrm{FF}}}{W}.
  \]
  The backward pass all-gathers the weights and then reduce-scatters the three gradients, so it costs
  \[
    \frac{12 (N_{\mathrm{FSDP}}-1) D D_{\mathrm{FF}}}{N_{\mathrm{FSDP}} W}.
  \]
\end{answer}

\subq{c} Scaling bounds for $N_{\mathrm{FSDP}}$ in forward and backward.
\deliverable{Two inequalities with $N_{\mathrm{FSDP}}$ on one side.}
\begin{answer}
  For the forward pass, requiring communication to fit under compute gives
  \[
    N_{\mathrm{FSDP}} \le 1 + \frac{B W}{C}.
  \]
  The backward pass gives the same bound,
  \[
    N_{\mathrm{FSDP}} \le 1 + \frac{B W}{C},
  \]
  because both the compute and communication costs are doubled relative to the forward/weight-gather comparison.
\end{answer}


\problem{tp\_calcs}{4}

\subq{a} Backward pass of the TP strategy in the handout, in terms of $\mathbf{dy}$, sharded weights, saved activations, and communication primitives; produce $\mathbf{dW}_1^{(i)}, \mathbf{dW}_2^{(i)}, \mathbf{dW}_3^{(i)}$, and $\mathbf{dx}$.
\deliverable{A series of equations describing the TP backward pass.}
\begin{answer}
\begin{align*}
  \mathbf{dz}^{(i)} &= \mathbf{dy} \left(\mathbf{W}_3^{(i)}\right)^\top \\
  \mathbf{dW}_3^{(i)} &= \left(\mathbf{z}^{(i)}\right)^\top \mathbf{dy} \\
  \mathbf{dx}_2^{(i)} &= \mathbf{dz}^{(i)} * f(\mathbf{x}_1^{(i)}) \\
  \mathbf{dx}_1^{(i)} &= \mathbf{dz}^{(i)} * f'(\mathbf{x}_1^{(i)}) * \mathbf{x}_2^{(i)} \\
  \mathbf{dW}_2^{(i)} &= \mathbf{x}^\top \mathbf{dx}_2^{(i)} \\
  \mathbf{dW}_1^{(i)} &= \mathbf{x}^\top \mathbf{dx}_1^{(i)} \\
  \mathbf{dx}^{(i)}_{\mathrm{local}} &=
    \mathbf{dx}_1^{(i)} \left(\mathbf{W}_1^{(i)}\right)^\top
    +
    \mathbf{dx}_2^{(i)} \left(\mathbf{W}_2^{(i)}\right)^\top \\
  \mathbf{dx} &= \mathrm{all\text{-}reduce}\left(\{\mathbf{dx}^{(i)}_{\mathrm{local}}\}_{i=0}^{N_{\mathrm{TP}}-1}\right).
\end{align*}
\end{answer}

\subq{b} Forward and backward FLOPs with $N_{\mathrm{TP}}$.
\deliverable{Two answers in terms of $B, D, D_{\mathrm{FF}}, N_{\mathrm{TP}}$.}
\begin{answer}
  The forward pass costs
  \[
    \frac{6 B D D_{\mathrm{FF}}}{N_{\mathrm{TP}}},
  \]
  because all three FFN matmuls are split across the tensor-parallel axis. The backward pass costs
  \[
    \frac{12 B D D_{\mathrm{FF}}}{N_{\mathrm{TP}}},
  \]
  since each backward matmul also uses only a \(1/N_{\mathrm{TP}}\) shard of the FFN hidden dimension.
\end{answer}

\subq{c} Forward and backward communication time with $N_{\mathrm{TP}}$.
\deliverable{Two answers in terms of a subset of $B, D, D_{\mathrm{FF}}, N_{\mathrm{TP}}, W$.}
\begin{answer}
  The forward pass has one all-reduce on the output activation of size \(BD\) FP16 elements, so
  \[
    \frac{4 (N_{\mathrm{TP}}-1) B D}{N_{\mathrm{TP}} W}.
  \]
  The backward pass has the same cost for the all-reduce of \(\mathbf{dx}\):
  \[
    \frac{4 (N_{\mathrm{TP}}-1) B D}{N_{\mathrm{TP}} W}.
  \]
\end{answer}

\subq{d} Scaling bounds for $N_{\mathrm{TP}}$ in forward and backward.
\deliverable{Two inequalities with $N_{\mathrm{TP}}$ on one side.}
\begin{answer}
  The forward pass remains compute-bound while
  \[
    N_{\mathrm{TP}} \le 1 + \frac{3 D_{\mathrm{FF}} W}{2C}.
  \]
  The backward pass has twice as much compute for the same activation all-reduce, so its bound is
  \[
    N_{\mathrm{TP}} \le 1 + \frac{3 D_{\mathrm{FF}} W}{C}.
  \]
\end{answer}


\problem{fsdp\_tp\_calcs}{6}

\subq{a} Forward-pass FLOPs with $N_{\mathrm{FSDP}}$ FSDP + $N_{\mathrm{TP}}$ TP.
\deliverable{Answer in terms of $B, D, D_{\mathrm{FF}}, N_{\mathrm{FSDP}}, N_{\mathrm{TP}}$.}
\begin{answer}
  The forward pass costs
  \[
    \frac{6 B D D_{\mathrm{FF}}}{N_{\mathrm{FSDP}} N_{\mathrm{TP}}}.
  \]
  FSDP splits the batch by \(N_{\mathrm{FSDP}}\), and TP splits each FFN matmul by \(N_{\mathrm{TP}}\).
\end{answer}

\subq{b} Forward-pass communication time assuming FSDP-axis and TP-axis collectives can be overlapped.
\deliverable{Answer in terms of a subset of $B, D, D_{\mathrm{FF}}, N_{\mathrm{FSDP}}, N_{\mathrm{TP}}, W$ (expected to be a max of two quantities).}
\begin{answer}
  With overlap, the forward communication time is
  \[
    \max\left(
      \frac{6 (N_{\mathrm{FSDP}}-1) D D_{\mathrm{FF}}}{N_{\mathrm{FSDP}} N_{\mathrm{TP}} W},
      \frac{4 (N_{\mathrm{TP}}-1) B D}{N_{\mathrm{FSDP}} N_{\mathrm{TP}} W}
    \right).
  \]
  The first term is the three FSDP weight all-gathers after TP sharding, and the second term is the TP output all-reduce on the batch shard.
\end{answer}

\subq{c} With overlap, how large can $N = N_{\mathrm{TP}} N_{\mathrm{FSDP}}$ grow before the forward pass is communication bottlenecked? Find the optimal split.
\deliverable{Inequality with $N$ on one side, plus justification.}
\begin{answer}
  Since the two communication axes overlap, each term only needs to be below the compute time
  \[
    \frac{6 B D D_{\mathrm{FF}}}{N_{\mathrm{FSDP}} N_{\mathrm{TP}} C}.
  \]
  This gives
  \[
    N_{\mathrm{FSDP}} \le 1 + \frac{B W}{C},
    \qquad
    N_{\mathrm{TP}} \le 1 + \frac{3 D_{\mathrm{FF}} W}{2C}.
  \]
  Thus the best split uses both axes up to their limits, so
  \[
    N \le
    \left(1 + \frac{B W}{C}\right)
    \left(1 + \frac{3 D_{\mathrm{FF}} W}{2C}\right).
  \]
\end{answer}

\subq{d} Same question but assuming no overlap between FSDP and TP collectives.
\deliverable{Inequality with $N$ on one side, plus justification.}
\begin{answer}
  Without overlap, the two communication terms add:
  \[
    \frac{6 (N_{\mathrm{FSDP}}-1) D D_{\mathrm{FF}}}{N W}
    +
    \frac{4 (N_{\mathrm{TP}}-1) B D}{N W}
    \le
    \frac{6 B D D_{\mathrm{FF}}}{N C}.
  \]
  Equivalently,
  \[
    3D_{\mathrm{FF}}(N_{\mathrm{FSDP}}-1)
    +
    2B(N_{\mathrm{TP}}-1)
    \le
    \frac{3 B D_{\mathrm{FF}} W}{C}.
  \]
  Optimizing this continuous split gives
  \[
    N_{\mathrm{FSDP}} =
    \frac{\frac{3 B D_{\mathrm{FF}} W}{C}+3D_{\mathrm{FF}}+2B}{6D_{\mathrm{FF}}},
    \qquad
    N_{\mathrm{TP}} =
    \frac{\frac{3 B D_{\mathrm{FF}} W}{C}+3D_{\mathrm{FF}}+2B}{4B},
  \]
  and therefore
  \[
    N \le
    \frac{\left(\frac{3 B D_{\mathrm{FF}} W}{C}+3D_{\mathrm{FF}}+2B\right)^2}
    {24 B D_{\mathrm{FF}}}.
  \]
\end{answer}
